[CD ANALYSIS]

KProtoPTA (K=3 global EMA prototypes, fixed) vs MPPTA v2 (patch-level):
  - EuroSAT: +4.35pp (56.91→61.26, nearly matches PTA 61.40)
  - UCF101:  +3.51pp (66.38→69.89)
  - Oxford Pets: +1.58pp, Oxford Flowers: +1.21pp
  - KProtoPTA avg = 64.44% vs MPPTA v2 avg = 63.39% (+1.05pp net)

Why KProtoPTA beats MPPTA on most datasets:
  - Uses global CLS embeddings (same space CLIP was trained in)
  - Preserves PTA's refinement formula: refined = normalize(alpha*text + (1-alpha)*proto)
  - Scores as max_k(img @ refined_k.T) — stays in aligned text-image subspace
  - PTA's soft update gate (all classes w >= 0.1) → more stable bank updates

Why KProtoPTA still trails PTA by 2.51pp avg:
  - With ~100 test samples/class, K=3 slots each get ~33 observations vs PTA's ~100
  - Noisier EMA estimate per slot; sparse signal distributed over K prototypes
  - max_k selection adds positive bias; may need re-calibrated 100× scalar

Known bug fixed in KProtoPTA (was causing catastrophic -20pp regression):
  - Initializing proto_banks to text_feat created t=0 logit doubling:
    final = clip_logits + 100 * (clip_logits/100) = 2 * clip_logits
  - Fix: initialize proto_banks to zeros (same as PTA's target_prototype init)
  - This ensures the proto correction starts at 0 and grows naturally with data

MPPTA v2 (patch-level) analysis:
  - Patch embeddings not semantically aligned with text (CLIP trains [CLS] token, not patches)
  - Greedy k-means on 196 patches adds noise; background patches corrupt prototypes
  - Results in -5.13pp avg below PTA despite extensive hyperparameter tuning

 Architecture Summary
==============================================================================

  Method      | Core mechanism                           | Config dir
  -------------------------------------------------------------------
  PTA         | Single global EMA prototype per class    | configs/
  MPPTA v2    | Patch-level k-means + cosine scoring     | configs_multi_proto/
  KProtoPTA   | K=3 global EMA protos, PTA formula       | configs_kproto/
              |   refined_k = normalize(α*text+(1-α)*p_k)|
              |   logit = clip + 100*max_k(img@refined_k)|

  Config keys (KProtoPTA):
    K:              3    — prototypes per class
    alpha:          0.01 — text anchor weight (same as PTA)
    T:              20.0 — EMA temperature (same as PTA)
    conf_threshold: adaptive per dataset (0.4–0.7)
