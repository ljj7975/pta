How to Use MultiProtoPTA Visualizer

📊 What is this?

This tool helps you inspect and understand how MultiProtoPTA adapts CLIP vision-language models at test time. It shows:

  • CLIP's zero-shot text embedding contribution
  • Prototype bank updates (learned patch-level visual features)
  • How prototypes are matched and scored for each class

🔄 Live Mode vs. Replay Mode

Live Mode (recommended for first-time use)
  1. Select a dataset (e.g., caltech101, eurosat)
  2. Pick a backbone model (ViT-B/16 or RN50)
  3. Set number of samples to process (200–500 for quick demo)
  4. Click Load
The system will:
  • Load CLIP and process samples one-by-one
  • Build prototype banks incrementally as it processes
  • Show you results as they happen
⏱️ May take several minutes depending on number of samples and dataset size.
Replay Mode (for offline inspection)
  1. Paste the path to a pre-saved `.pkl` file (e.g., outputs/viz/caltech101_n200.pkl)
  2. Click Load
You can then navigate and explore results instantly without re-processing.
Useful for sharing results or inspecting past runs.

🎯 Understanding the Display

  • Sample Image: The test input image
  • Prediction Panel: Ground truth, prediction result, and running accuracy
  • Score Bars: Visual bar chart showing final logits for top 10 classes
  • Class Details: Expand any class to see scores, prototypes, and active matches
  • Top Matches: Global ranking of all prototypes by patch cosine similarity

🧠 Understanding the Scores

Text Score
How well the CLIP text embedding matches this image (zero-shot baseline). Range: ~[0–100].
Raw Proto
Evidence-weighted average similarity of the top matching patches to this class's prototype bank. Range: [0–1].
Centered Proto (per-class)
The raw_proto score adjusted by subtracting the mean raw_proto across all 1000 classes. This is a per-class aggregation.
Formula: centered_proto = raw_proto − mean(raw_proto for all classes)
Why? Prevents the model from being biased toward any single class. Keeps logits balanced.
Per-prototype note: Each active prototype also has its own centered_score computed the same way. The class-level centered_proto is the aggregation across all that class's prototypes.
Final Logit
text_score + τ × centered_proto — the number used for prediction (class with highest logit wins).
Softmax Prob (%)
Probability of this class from softmax(all logits). Shows model confidence.

💾 Understanding the Prototype Bank

Bank Size (K)
Number of prototypes stored for this class. This grows over time!
  • Samples 1–50: K usually 1–3 (algorithm just starting, learning first patches)
  • Samples 100–200: K grows to 5–15 as more matching patches are collected
  • Samples 200+: K approaches max_K (default 30 per class)
→ This is normal and expected! MultiProtoPTA adapts incrementally over time.
Active Prototypes (shown in class details)
Prototypes from this class that actually matched the current image.
  • If Bank Size = 10 but only 1 active prototype: Only 1 of 10 stored prototypes had high similarity to this image's patches
  • The other 9 learned from different visual patterns, so they don't match this image
  • → This is selective and correct! The model only uses relevant prototypes.
Appearance (App)
How many images this prototype has matched with so far. Higher = more robust prototype learned from diverse examples.
Evidence Weight (Evid_W)
Confidence weight for this prototype: how much to trust it.
Formula: min(1.0, 0.5 + 0.5 × (K−1) / 4)
Ranges 0.5 → 1.0. Older, well-tested prototypes have weight 1.0; new ones have weight 0.5–0.75.
